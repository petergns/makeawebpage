Important Concepts

Lesson 1: The Basics of the Web and HTML

How the Web Works

Computers connect together, in what is known as the web or world wide web. When you visit a web page such as udacity.com, your computer sends a request for information to a 'Server', which acts kind of like a phone book directing people to each other. This 'Server' sends information back and forth to your browser, and allows you to view the web page.

HTML

HTML or Hypertext Markup language is what we use to view and explore the web. This takes the form of HTML documents, which contain text, and a 'Markup' that determines what the content of the page will look like once it is loaded.

Tags and Elements

HTML documents are made up of Elements. We use Tags to tell the browser what elements will be used and in what way. If there is text on a web page for example, then you might like to use a bold tag on some important words, use the italics or em tag, or perhaps create a break between words or sentances with a
break tag.

Why Computers are Stupid

Because when you put content into a computer, it will interpret exactly what you put in, even if it means that a program won't load correctly, which is why it is important to make sure that tags and elements of a program are correct before you ask a computer to load them.

Inline vs Block Elements

HTML elements fall into two categories, either block or inline elements. Block elements can take up the full width of a page, whereas inline elements cannot. For example, a paragraph or p tag creates a invisible box around content. Inline elements such as the span tag create an invisible inline box tightly around content.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

Using Developer Tools on Firefox or Chrome, you can look at what makes a web page, in terms of how the HTML document is set up, and make modifications without having to make the changes directly on the web page itself.

Using Developer Tools the tags can be altered, such as border-radius or color, to see what a change on the web page would do. This web page uses both border-radius and color, so it was helpful to use Developer Tools to test out what the page would look like, when I was putting it together.

HTML's "tree-like" Structure

HTML documents are built much like a "tree", with HTML code branching out from each other, with child elements being the smaller branches, and the larger branches being parent elements. You can view the "tree" structure, and follow the branches from the body of the HTML, to smaller branches such as the divs, classes, and paragraph or P tags.

Indentations and Boxes

When you look at an HTML page on a text-editor, you can see indentations, which go further and further in, this shows which element tags correspond to what content boxes. The further in the indentation, the more nested the box would be in that HTML document.

Text Editors

Programmers use Text Editors to write and organize their code, which enables them to develop codes quicker through use of the features that are provided. Through the use of Text-Editors like Atom, Sublime Text, and Notepad++ you can enable auto-formatting, and see codes easier than if using something basic like word or notepad or writing by hand.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repeating Code and Errors

It is important while programming to not let areas slip into your code, as the computer will read those errors and not correct them when it loads your code.

When the browser for example loads a web page, it will process the page as you have coded it, which means that if you don't add the correct tags or close them correctly, the page will be formatted not as you would want it to be formatted.

Another problem is that you might repeat code, and thus make the code longer to process than it needs to, which will reduce the speed of an app or web page.

It also makes it harder to edit a page, as it is easier to edit a single div that covers multiple areas, than having multiple divs that do the same thing that you have to edit individually.

CSS

CSS or Cascading Style Sheets are much like the paint or style of a house, as you are adding aspects to your app or web page, that make it look more pleasing to the eye, or improve the user experience.

When you code with CSS and HTML, you can include a CSS style page to add color, different fonts, and more major changes, often through use of classes, which can be added to divs and other page tag elements.
